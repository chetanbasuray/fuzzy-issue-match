# AGENTS.md

This file helps coding agents quickly contribute to `fuzzy-issue-match`.

## Mission

Build and maintain an npm library that detects duplicate GitHub issues using fuzzy and semantic matching.

## Current State

`v0.1.0` is a foundation release with no production matching logic yet.
Roadmap items are tracked in GitHub Issues.

## Agent Contribution Guidelines

1. Start by checking open roadmap issues and labels.
2. Prefer small PRs that map to one issue.
3. Add or update tests for every behavior change.
4. Use conventional commits.
5. Document assumptions and tradeoffs in PR descriptions.

## Priority Labels

- `roadmap`
- `good first issue`
- `help wanted`
- `agent-friendly`
- `community`
- `security`
- `performance`

## Definition of Done

1. CI is green.
2. Security checks are green.
3. Docs are updated if behavior changed.
4. PR reviewed and approved before merge (admins may override when appropriate).
